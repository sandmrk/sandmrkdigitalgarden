新视频上线——AI 编程工具都在用 Agent Skills：一键安装 + 推荐清单 + 编写最佳实践 整理了一份 Skills 指南： ✓ 主流工具配置路径对照 ✓ 一键安装工具推荐 ✓ 真正值得用的 Skills 清单 ✓ Anthropic 官方最佳实践提炼 你写的 Skill 是「能跑」还是「好用」？ 看了 Anthropic 官方的最佳实践后发现，差距主要在这几点： 简洁——Claude 已经很聪明，别解释它知道的东西 自由度——脆弱操作给死指令，灵活任务给方向 反馈循环——跑完要验证，出错要能自动修

### 🧵 深度展开
> [!NOTE] 补充 1
> 新视频上线——我怎么用 Moltbot（原 Clawdbot）：真实体验与进阶技巧 这期我将介绍 - Skills 生态怎么用（ClawdHub / Awesome Skills） - 多任务并行：子代理怎么跑、Telegram 并发/队列怎么理解 - 常用 Telegram 指令 + Mac 双机协作（屏幕共享/文件传输） 如果有条件，建议直接在 Mac 上使用而不是 VPS 上使用 Moltbot
> [!NOTE] 补充 2
> 刚把 Clawdbot 迁移到 Moltbot ,一看仓库，又改名了，现在是 openclaw
> [!NOTE] 补充 3
> 新视频上线——开源最强？实测Kimi K2.5：多模态/Agent集群/编码，优缺点一并讲到 “一个 Agent 不够用，那就来一群。” Kimi 的 Agent 集群在官网 Beta 里已经能一次拉 4 个一起工作。（可以分多个子 Agent，但目前并行是 4个） Kimi 这次还重点提了 Kimi Code：支持图像/视频作为输入，并且有 VSCode 插件生态。 官方展示里有个我第一次看到的用法：对 Word 文档批量加注释。 Kimi 的审美真得很好，Agent 界面做得太优雅了，前端页面设计能力逼近 Gemini 3 Pro。
> [!NOTE] 补充 4
> 详解 & 实测开源TTS天花板Qwen3-TTS ：5个模型 + 10种语言 + 3秒音色克隆（含 Mac部署） 如果你喜欢 Gemini 3 的“指令驱动TTS”，那 Qwen3-TTS 你大概率也会爱 一句自然语言提示就能驱动语音生成，而且开源、模型选择还多 我着重讲了如何进行声音控制，值得看下 此外，我用 MLX 版本跑 Qwen3-TTS 昨天踩了不少坑， 但也找到可用解法（把默认 CUDA 改 MPS、注意力改 SDPA 等）。
> [!NOTE] 补充 5
> 实测升级版MiniMax Agent：桌面端+专家Agents值得一试！Win/Mac都能用 MiniMax Agent 这次升级确实有料： 它把 Cowork 那套“桌面端工作空间”做出来了，还补齐了 Windows 支持； 同时网页端新增 专家 Agents，可以按场景像选“专家同事”一样调用。 我最喜欢的一点是：可复核。 你能直接看到它的工作流、产出的中间文件、以及调用了哪些子 Agent，透明度比 Cowork 更高。 它不是“黑箱给你一个结果”，更像“同事交付一套可审计的方案”。
